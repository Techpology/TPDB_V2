#100 I_Database

#101 TP_FAILED_CREATETPDATABASE(Char *, Char *)

Function to allocate a new Database.
Standard initiation function for "TP_Database" has failed.
Args:
- (Char*) This acts as a enforced obsolete value. It is in place to allow for future updates or plugins where one can
hold multiple databases. Preferable to provide a proper name.
- (Char*) The path/name to be provided to the database which will also be the name of the folder acting as root for the database.

Notes:
- Make sure you are passing a constant string. ex: "hello".
If you need to pass an allocated string, make sure to free it after

There are no more known reasons or bugs within this function, if it fails, report to the git repository.

#200 I_Table

#201 TP_FAILED_CREATETPTABLE(Char *, TPDatabase *)

Function to allocate a new table within target Database.
Standard initiation function for "TP_Table" has failed.
Args:
- (Char*) The name to be provided to the table which will also be the name of the folder holding the tables rows
- (TPDatabase*) Target Database to hold tables and so on. Make sure to allocate the TPDatabase beforehand properly.

Notes:
- Make sure you are passing a constant string. ex: "hello".
If you need to pass an allocated string, make sure to free it after

- There can be an issue with TP_StrnCat, make sure you have assigned a proper path to your database, no '/' in the end

There are no more known reasons or bugs within this function, if it fails, report to the git repository.

#500 StringTools

#501 TP_FAILED_STRNCAT(Char *, Int, ...)

Function to dynamically concatonate multiple strings together.
Args:
- (Char*) The string which you want to be in the begining of the resulting string.
- (Int) How many strings will you be providing to concatonate to base string.
- (Char*) {n} Strings to concatonate.

Notes:
- Make sure you are passing a constant string. ex: "hello".
If you need to pass an allocated string, make sure to free it after

- Make sure that the first argument (Char* _baseString) is not NULL.
If you want to replicate passing NULL, then use an empty string ("").

There are no more known reasons or bugs within this function, if it fails, report to the git repository.
